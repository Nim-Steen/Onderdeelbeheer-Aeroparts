# Onderdeelbeheer-Aeroparts
Calco project waarin we buggy Python-code voor het bestellen van vliegtuigonderdelen zullen fixen en testen.

## Nieuw met GitHub?
1. Download GitHub Desktop via de Company Portal
2. Maak een account aan - Dit zal de naam zijn die we bij jouw werk zien
3. Klik op https://github.com/Nim-Steen/Onderdeelbeheer-Aeroparts op Code -> Open with GitHub Desktop
4. Klik op Clone (kies eventueel een andere plek om het op je laptop te downloaden, maar kies hier niet de SharePoint!)

## Workflow (Team rules)
Voor alle code wordt de volgende werkwijze gehanteerd:
*	Werk altijd in een aparte branch (bijv. feature/..., bugfix/...)
*	Maak een Pull Request (PR) voor elke wijziging
*	Voeg een korte en duidelijke beschrijving toe aan de PR
*	Minimaal één teamlid voert een review uit voordat de code wordt gemerged

Doel: Kwaliteit waarborgen en samenwerking verbeteren.

## Bugs
In de SharePoint staat een Excel met de testcases, waarin ook een tabblad is om risks en issues bij te houden. 
Het is echter handig om deze (ook) in GitHub bij te houden, dus, wanneer je (met of zonder test) een bug vindt:
1. Ga in GitHub naar Issues
2. Klik op [New Issue]
3. Kies "Bug report"
4. Vul het template in, haal kopjes weg die niet relevant zijn

Op deze manier kunnen we nieuwe branches aanmaken voor bugs, en pull requests aan bugs koppelen om overzicht te houden waar iets gefixt wordt/is.
